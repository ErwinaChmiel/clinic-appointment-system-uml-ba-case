# Requirements

## Business requirements

| ID | Requirement |
|---|---|
| BRQ-01 | The system shall allow patients to book appointments online. |
| BRQ-02 | The system shall reduce manual confirmations performed by reception staff. |
| BRQ-03 | The system shall support prepayments for selected services. |
| BRQ-04 | The system shall provide an unambiguous status for each appointment. |

## Functional requirements

| ID | Requirement |
|---|---|
| FR-01 | The patient can search for a specialist by specialization. |
| FR-02 | The patient can view available slots for a specialist. |
| FR-03 | The system temporarily locks the selected slot during booking. |
| FR-04 | The system checks whether a selected service requires prepayment. |
| FR-05 | The system sends payment authorization to the payment gateway. |
| FR-06 | The system confirms the appointment after booking conditions are met. |
| FR-07 | The system sends an appointment confirmation message. |
| FR-08 | The patient can cancel an appointment according to business rules. |

## Non-functional requirements

| ID | Requirement |
|---|---|
| NFR-01 | The booking process should be completed in less than 3 minutes. |
| NFR-02 | The system should be available at least 99.5% of monthly time. |
| NFR-03 | Patient data should be protected according to privacy and security requirements. |
| NFR-04 | The interface should be responsive and usable on mobile devices. |
