# Business context — MediVisit

## Current situation

The medical provider handles appointment booking through multiple channels: phone, e-mail and reception desk. There is no single source of truth for specialist availability and appointment status.

## Problem

Key operational problems:

- double-booking of the same slots,
- manual appointment confirmations,
- no automated prepayment handling,
- limited visibility of appointment status for patients,
- high number of reception desk calls.

## Business objective

The objective is to introduce an online solution that allows patients to choose a specialist, select an available slot and receive appointment confirmation without reception desk involvement.

## Expected benefits

| Benefit | Description |
|---|---|
| Reduced manual work | Fewer phone calls and manual confirmations. |
| Better information availability | Patients see current availability online. |
| Fewer errors | The system locks a slot during booking. |
| Better payment control | Selected services require prepayment before confirmation. |
