# Backlog — user stories

| ID | User story | Acceptance criteria |
|---|---|---|
| US-01 | Jako pacjent chcę wyszukać dostępne terminy, aby wybrać dogodną wizytę. | Lista terminów pokazuje lekarza, usługę, datę i godzinę. |
| US-02 | Jako pacjent chcę zarezerwować wybrany termin, aby mieć potwierdzoną wizytę. | Termin jest tymczasowo blokowany, a następnie potwierdzany po spełnieniu warunków. |
| US-03 | Jako pacjent chcę opłacić przedpłatę online, aby zakończyć rezerwację usługi wymagającej płatności. | Po pozytywnej płatności wizyta otrzymuje status potwierdzony. |
| US-04 | Jako recepcjonistka chcę zarządzać wizytami, aby obsługiwać wyjątki i anulacje. | Recepcja może wyszukać, zmienić lub anulować wizytę zgodnie z uprawnieniami. |
| US-05 | Jako lekarz chcę zarządzać dostępnością, aby pacjenci widzieli aktualne terminy. | Zmiana dostępności wpływa na listę dostępnych terminów. |
